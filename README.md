Architecture Overview
The CNN-BiLSTM model (Algorithm 1, Section III.C) integrates:
вЂў CNN Stage: 3 Conv1D layers (64в†’128в†’256 filters, kernel=3) + ELU + AvgPool
вЂў BiLSTM Stage: 2 bidirectional LSTM layers (128 units each)
вЂў Dual Output: Classification (Softmax) + Morphology Prediction (Linear, 47 PCA coefficients)
вЂў Total Parameters: 952,507 (Table IV)

Key Design Choices
вЂў Sequence length: 16 consecutive spikes (Section III.C.2)
вЂў Zero-padding: Left for first 15 spikes, right for final spikes
вЂў Prediction scope: Single-step forecast of next detected spikeвЂ™s morphology
вЂў Prediction invalidation: If ISI > 500ms (Section III.C.3)
вЂў Value clipping: [-5, +5] standard deviations during training

Known Limitations
From Section VI.B:
1. Computational Cost: 2.3Г—10пїї FLOPs вЂ” requires GPU acceleration
2. Predictive Limitation: Predicts morphology (PCA coefficients) only, not ISI or spike timing
3. Cross-Dataset Generalization: 12.2% drop (motor cortex в†’ hippocampus), 14.7% drop (rodent
в†’ primate)
4. Supervision Dependency: 96.4% accuracy requires full labels; 10% labels yield 88.9%
5. Single-Electrode Processing: 12% over-splitting for multi-electrode recordings
6. Reproducibility: Proprietary MEA dataset restrictions (see Step 2 in remediation plan)
